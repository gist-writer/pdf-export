# T1 — Fix `stripLinks` + restore concurrency lock

**Scope:** `src/main.ts` only.  
**Depends on:** nothing — standalone fixes.  
**Risk:** Low — fixes two existing bugs, no new behaviour added.  
**Verify:** Export a document containing `[text](url)` links and `![alt](url)` images. Links should render as plain text (unchanged). Image lines should survive intact and appear as raw `![alt](url)` strings in the output (they will be handled in T3). Trigger two rapid PDF exports — the second should be silently ignored.

---

## What to change

### 1. Fix image lines — detect before `stripLinks` runs

`stripLinks` itself is **not changed**. Instead, in the line parser loop, image lines are detected and skipped before `stripLinks` ever runs on them. A line is an image if it matches `/^!\[.*\]\(.*\)$/`.

In the `for (const line of lines)` loop, add this guard **before** any call to `stripLinks`:

```ts
// Skip image lines — handled in T3
if (/^!\[.*\]\(.*\)$/.test(line)) {
  content.push({ text: line }); // placeholder — T3 replaces this
  continue;
}
```

This means:
- `![alt](url)` lines are never passed to `stripLinks` — they survive intact as raw strings
- `[text](url)` links in all other lines continue to be stripped to `text` as before
- `stripLinks` is unchanged — simpler, no regex surgery

### 2. Restore `pdfPending` boolean lock

The `pdfPending` concurrency guard was removed in a previous refactor. Making the message handler async (T3) re-opens the race condition. Add it back now before the handler becomes async.

```ts
// Add at module scope, near the top of main.ts:
let pdfPending = false;
```

The lock will be wired into the async handler in T3. Declaring it here keeps the diff clean.
