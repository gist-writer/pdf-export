# T1 — Fix `stripLinks` + restore concurrency lock

**Scope:** `src/main.ts` only.  
**Depends on:** nothing — standalone fixes.  
**Risk:** Low — fixes two existing bugs, no new behaviour added.  
**Verify:** Export a document containing `[text](url)` links and `![alt](url)` images. Links should render as plain text (unchanged). Image lines should survive `stripLinks` intact and appear as raw `![alt](url)` strings in the output (they will be handled in T3). Trigger two rapid PDF exports — the second should be silently ignored.

---

## What to change

### 1. Fix `stripLinks` — add negative lookbehind for `!`

The current regex matches both `[text](url)` and `![alt](url)`, stripping image syntax before any line parsing happens.

```ts
// Before:
function stripLinks(text: string): string {
  return text.replace(/\[([^\]]+)\]\([^)]*\)/g, '$1');
}

// After:
function stripLinks(text: string): string {
  return text.replace(/(?<!!)\[([^\]]+)\]\([^)]*\)/g, '$1');
}
```

The negative lookbehind `(?<!!)` ensures `![alt](url)` is not matched. `[text](url)` continues to be stripped to `text` as before.

### 2. Restore `pdfPending` boolean lock

The `pdfPending` concurrency guard was removed in a previous refactor. Making the message handler async (T3) re-opens the race condition. Add it back now before the handler becomes async.

```ts
// Add at module scope, near the top of main.ts:
let pdfPending = false;
```

The lock will be wired into the async handler in T3. Declaring it here keeps the diff clean.
