# T3 — Wire image fetching into `markdownToDocDef`

**Scope:** `src/main.ts` only.  
**Depends on:** T1 (stripLinks fix, pdfPending declared), T2 (`fetchImageAsBase64` present).  
**Risk:** Medium — `markdownToDocDef` becomes async, message handler becomes async, concurrency lock wired in. Core render pipeline change.  
**Verify:** Export a document with an Imgur image — image appears in PDF at full width. Export with a CORS-blocked image URL — italic alt text appears, PDF downloads successfully. Trigger two rapid exports — second is blocked. Export a document with only text — unchanged output.

## What to change

### 1. Make `markdownToDocDef` async

```ts
// Before:
function markdownToDocDef(markdown: string): object {

// After:
async function markdownToDocDef(markdown: string): Promise<object> {
```

### 2. Detect image lines before `stripLinks` runs

Inside the line-parsing loop in `markdownToDocDef`, add image detection as the **first** branch — before any call to `stripLinks`:

```ts
for (const rawLine of lines) {
  const line = rawLine.trim();

  // Image — must be checked before stripLinks
  const imgMatch = line.match(/^!\[(.*)\]\((.+)\)$/);
  if (imgMatch) {
    const [, alt, url] = imgMatch;
    const dataUri = await fetchImageAsBase64(url);
    if (dataUri) {
      content.push({ image: dataUri, width: 435 });
    } else {
      content.push({ text: alt || url, italics: true });
    }
    continue;
  }

  // All other line types — stripLinks safe to call now
  const stripped = stripLinks(line);
  // ... rest of existing parsing logic
}
```

### 3. Wire `pdfPending` into the message handler

The handler must become async and guard with `pdfPending`:

```ts
window.addEventListener('message', async (event: MessageEvent) => {
  if (event.origin !== 'https://gist-writer.github.io') return;
  if (event.data?.type !== 'EXPORT_PDF') return;

  if (pdfPending) return;
  pdfPending = true;

  try {
    const { filename, markdown } = event.data;
    const docDef = await markdownToDocDef(markdown);
    pdfMake.createPdf(docDef).download(filename);
  } finally {
    pdfPending = false;
    event.source?.postMessage(
      { type: 'EXPORT_PDF_DONE' },
      { targetOrigin: event.origin },
    );
  }
});
```

Using `try/finally` ensures `pdfPending` is always reset and `EXPORT_PDF_DONE` is always sent — even if `markdownToDocDef` throws unexpectedly.

---

## What NOT to change

- Do not modify `fetchImageAsBase64()` or `blobToBase64()`.
- Do not change `stripLinks()` — already fixed in T1.
- Do not touch `CORS_PROXIES` array.
- Inline images (`some text ![x](url) more text`) remain out of scope — the regex `/^!\[.*\]\(.*\)$/` only matches lines where the image is the entire line.

## Definition of done

- `markdownToDocDef` is `async`, returns `Promise<object>`.
- Image lines detected before `stripLinks` runs.
- Successful fetch → `{ image: dataUri, width: 435 }` node in PDF.
- Failed fetch → `{ text: alt, italics: true }` node in PDF.
- `pdfPending` lock prevents concurrent exports.
- `EXPORT_PDF_DONE` sent on all exit paths including error.
- Export with Imgur image → image appears in PDF.
- Export with CORS-blocked image → italic alt text, PDF downloads.
- Export with no images → output identical to before.
- No TypeScript errors.
- No new npm dependencies.
