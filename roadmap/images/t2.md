## What to change

### Add `fetchImageAsBase64()` to `src/main.ts`

```ts
const CORS_PROXIES = [
  'https://api.allorigins.win/raw?url=',
  'https://corsproxy.org/?',
  'https://thingproxy.freeboard.io/fetch/',
];

async function fetchImageAsBase64(url: string): Promise<string | null> {
  const attempts = [url, ...CORS_PROXIES.map(p => p + encodeURIComponent(url))];

  for (const attempt of attempts) {
    try {
      const res = await fetch(attempt, { signal: AbortSignal.timeout(5000) });
      if (!res.ok) continue;
      const blob = await res.blob();
      const mime = blob.type || 'image/jpeg';
      const b64 = await blobToBase64(blob);
      return `data:${mime};base64,${b64.split(',')[1]}`;
    } catch {
      continue;
    }
  }

  return null;
}

function blobToBase64(blob: Blob): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => resolve(reader.result as string);
    reader.onerror = reject;
    reader.readAsDataURL(blob);
  });
}
```

### Fetch strategy

1. **Direct fetch** — works for Imgur, `raw.githubusercontent.com`, and any CORS-open host
2. **`allorigins.win`** — open-source proxy, passes all content types including binary
3. **`corsproxy.org`** — independent fallback (different service from `corsproxy.io`)
4. **`thingproxy.freeboard.io`** — third fallback
5. All failed → return `null` → caller renders italic alt text

Each attempt has a 5-second timeout via `AbortSignal.timeout(5000)`. Worst-case with all four attempts timing out: 20 seconds before `null` is returned. In practice, direct fetch succeeds instantly for Imgur and GitHub-hosted images.
