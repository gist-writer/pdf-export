# T4 — Type Safety Improvements

Status: `done`

## Goal

After this tranche, all `as unknown as Content` casts are removed, the `postMessage` listener has runtime type validation, and shared types use `interface` instead of inline type aliases. The code is now type-safe without escape hatches.

## Steps

### 1. Replace `as unknown as Content` with proper pdfmake list types

Import `ContentOrderedList` and `ContentUnorderedList` from pdfmake and use them directly instead of casting through `unknown`.

```ts
import type {
  Content,
  ContentOrderedList,
  ContentUnorderedList,
  TDocumentDefinitions,
  TFontDictionary,
} from 'pdfmake/interfaces';

const flushList = () => {
  if (!pendingList) return;
  if (pendingList.type === 'ul') {
    content.push({ ul: pendingList.items, margin: [0, 0, 0, 6] } as ContentUnorderedList);
  } else {
    content.push({ ol: pendingList.items, margin: [0, 0, 0, 6] } as ContentOrderedList);
  }
  pendingList = null;
};
```

### 2. Add runtime message validation with a type guard

Replace the loose destructuring + truthy checks with a proper type guard function.

```ts
interface ExportPdfMessage {
  type: 'EXPORT_PDF';
  filename: string;
  markdown: string;
}

function isExportPdfMessage(data: unknown): data is ExportPdfMessage {
  return (
    typeof data === 'object' &&
    data !== null &&
    'type' in data &&
    (data as Record<string, unknown>).type === 'EXPORT_PDF' &&
    typeof (data as Record<string, unknown>).filename === 'string' &&
    typeof (data as Record<string, unknown>).markdown === 'string' &&
    (data as Record<string, unknown>).filename !== '' &&
    (data as Record<string, unknown>).markdown !== ''
  );
}
```

Update the message handler:

```ts
window.addEventListener('message', async (event) => {
  if (event.origin !== ALLOWED_ORIGIN) return;
  if (!isExportPdfMessage(event.data)) return;
  const { filename, markdown } = event.data;
  // ...
});
```

### 3. Convert `InlineNode` from type alias to interface

```ts
// Before:
type InlineNode = { text: string; bold?: boolean; italics?: boolean; font?: string };

// After:
interface InlineNode {
  text: string;
  bold?: boolean;
  italics?: boolean;
  font?: string;
}
```

### 4. Add `__APP_VERSION__` declaration

Add the Vite-defined global to `src/vite-env.d.ts`:

```ts
declare const __APP_VERSION__: string;
```
