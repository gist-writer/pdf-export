# T5 — Parser Refactor: Headings, Lists, Inline Markup

## Goal

After this tranche, the heading parser uses a single regex instead of six repetitive `if/else if` branches, list item parsing handles variable whitespace, and the inline parser supports `***bold italic***` syntax. These are correctness and robustness improvements to the parsing logic.

## Steps

### 1. Replace repetitive heading parsing with regex

The current code has six near-identical branches that hard-code slice offsets. Replace with a single regex match.

```ts
// Before: 6 if/else if branches with hard-coded offsets
if (line.startsWith('###### ')) content.push({ text: parseInline(stripLinks(line.slice(7))), style: 'h6' });
else if (line.startsWith('##### ')) ...

// After: single regex
const headingMatch = line.match(/^(#{1,6})\s+(.*)$/);
if (headingMatch) {
  const level = headingMatch[1].length;
  content.push({ text: parseInline(stripLinks(headingMatch[2])), style: `h${level}` });
}
```

### 2. Fix unordered list item parsing

Replace `line.slice(2)` (assumes single-char marker + one space) with a regex that handles variable whitespace and all three markers (`-`, `*`, `+`).

```ts
// Before:
pendingList.items.push({ text: parseInline(stripLinks(line.slice(2))) });

// After:
pendingList.items.push({ text: parseInline(stripLinks(line.replace(/^[-*+]\s+/, ''))) });
```

### 3. Add `***bold italic***` support to `parseInline`

Extend the inline regex to handle triple-asterisk markup.

```ts
// Before:
const re = /`([^`]+)`|\*\*(.+?)\*\*|\*(.+?)\*/g;

// After:
const re = /`([^`]+)`|\*\*\*(.+?)\*\*\*|\*\*(.+?)\*\*|\*(.+?)\*/g;
```

Update the match handler to handle the new capture group (index shifts by one):

```ts
if (m[1] !== undefined) nodes.push({ text: m[1], font: 'iAWriterMono' });        // inline code
else if (m[2] !== undefined) nodes.push({ text: m[2], bold: true, italics: true }); // bold+italic
else if (m[3] !== undefined) nodes.push({ text: m[3], bold: true });                // bold
else if (m[4] !== undefined) nodes.push({ text: m[4], italics: true });             // italic
```

### 4. Hoist regex constants

Move the list-detection regexes out of the loop to module scope.

```ts
const UL_RE = /^[-*+] /;
const OL_RE = /^\d+\. /;
```
