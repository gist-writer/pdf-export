# T3 — Deploy Script Fixes + Linting Setup

Status: `done`

## Goal

After this tranche, the deploy script follows standard semver (no artificial rollover at 10) and works on both macOS and Linux (CI-compatible). The project also has ESLint and Prettier configured, catching code style issues going forward.

## Steps

### 1. Fix non-standard semver rollover in `deploy.sh`

The script currently rolls patch → minor at 10 and minor → major at 10. Standard semver segments are unlimited.

Remove the rollover logic:

```bash
# Before:
PATCH=$((PATCH + 1))
if [ "$PATCH" -ge 10 ]; then
  PATCH=0
  MINOR=$((MINOR + 1))
fi
if [ "$MINOR" -ge 10 ]; then
  MINOR=0
  MAJOR=$((MAJOR + 1))
fi

# After:
PATCH=$((PATCH + 1))
```

### 2. Fix macOS-specific `sed` syntax

Replace `sed -i ''` (macOS-only) with `npm version` which is portable and already handles `package.json` version updates:

```bash
# Before:
sed -i '' "s/\"version\": \".*\"/\"version\": \"${NEW_VERSION}\"/" package.json

# After:
npm version "$NEW_VERSION" --no-git-tag-version
```

### 3. Install and configure ESLint + Prettier

```bash
npm install -D eslint @typescript-eslint/eslint-plugin @typescript-eslint/parser prettier
```

Create `.eslintrc.cjs`:

```js
module.exports = {
  root: true,
  parser: '@typescript-eslint/parser',
  plugins: ['@typescript-eslint'],
  extends: [
    'eslint:recommended',
    'plugin:@typescript-eslint/recommended',
  ],
  parserOptions: {
    ecmaVersion: 'latest',
    sourceType: 'module',
  },
  rules: {
    '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
  },
};
```

Create `.prettierrc`:

```json
{
  "singleQuote": true,
  "trailingComma": "all",
  "printWidth": 100,
  "semi": true
}
```

Add scripts to `package.json`:

```json
{
  "scripts": {
    "lint": "eslint src --ext .ts",
    "lint:fix": "eslint src --ext .ts --fix",
    "format": "prettier --write src"
  }
}
```

### 4. Fix existing lint errors

Run `npm run lint` and fix any issues it surfaces. Do not change logic — only fix lint-level issues (unused imports, missing types, etc.). Substantive code changes belong in later tranches.

### 5. Verify

- Run `deploy.sh` and confirm the version increments correctly past `.9` (e.g., `0.5.9` → `0.5.10`, not `0.6.0`).
- Run `deploy.sh` on a Linux environment (or CI) — confirm `npm version` works without the macOS `sed` issue.
- `npm run lint` exits cleanly with no errors.
